발표 진행 순서

  1. 문제 제시: 좌석 1000개, 접속자 10000명 상황을 설명한다.
  2. 수동 시연: S0001을 HELD로 만들고, 같은 좌석 중복 요청 실패를 보여준다.
  3. TTL 만료: 결제가 없으면 자동으로 AVAILABLE로 돌아가는 것을 보여준다.
  4. DB 확정: 결제 성공 후 CONFIRMED와 DB row 생성을 보여준다.
  5. 러시 시뮬레이션: 10,000명 러시로 대량 경쟁 제어를 강조한다.
서버와 데모 상태를 불러오는 중입니다.
